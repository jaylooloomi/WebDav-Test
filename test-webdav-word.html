📄 WebDAV Word 文件開啟測試

透過 ms-word:ofe 協議開啟 WebDAV 上的 Word 文件

ℹ️ 此頁面用於測試 WebDAV 上 Word 文件的開啟和同步修改功能。 請確保已完成以下準備：
  • IIS WebDAV 服務已啟用
  • Nginx 反向代理已配置
  • Windows 已安裝 Microsoft Word
  • 網路連接正常

🏠 本機測試（Localhost:8081）

直接透過 IIS WebDAV 服務開啟文件

▶ 用 Word 開啟 (localhost:8081)
ms-word:ofe|u|http://localhost:8081/2023_002.docx

🔀 Nginx 反代測試（Localhost:88）

透過 Nginx 反向代理 (Port 88) 開啟文件

▶ 用 Word 開啟 (localhost:88)
ms-word:ofe|u|http://localhost:88/WebDav/2023_002.docx

🌐 IP 地址測試（192.168.2.76:88）

透過實際主機 IP 地址開啟文件（用於跨機器測試）

▶ 用 Word 開啟 (192.168.2.76:88)
ms-word:ofe|u|http://192.168.2.76:88/WebDav/2023_002.docx

注意：此連結用於本機或同一網段上的其他電腦存取。

🧪 測試流程

  1. 點擊上方連結 → Word 應該開啟文件
  2. 修改內容 → 改動文件中的文字或格式
  3. 保存文件 → Ctrl+S 或 File > Save
  4. 檢查同步 →
    • 查看 IIS 日誌（通常在 C:\inetpub\logs\LogFiles）
    • 確認是否有 PUT/LOCK 請求
    • 下載文件對比，確認修改已保存
⚠️ 常見問題
  • 連結無法開啟： 確保 Word 已安裝，或在防火牆允許外連；
  • 認證失敗： 檢查 IIS 是否啟用了認證（匿名或 Windows）；
  • 檔案鎖定提示： 正常行為，表示 WebDAV LOCK 生效；
  • 修改未同步： 檢查 Nginx 是否正確轉發 WebDAV 方法。

🔍 立即驗證 WebDAV 狀態

在 PowerShell 中執行下列命令驗證 WebDAV 服務：

PowerShell -NoProfile -ExecutionPolicy Bypass -Command "cd D:\git\webdav; .\02-verify-webdav.ps1"

或直接執行：D:\git\webdav\02-verify-webdav.ps1
